Oasis-500M Multi-Chip Tensor Parallelism Plan
==============================================

GOAL: Scale DiT inference across N chips (initially 4) using tensor parallelism.
Target: 4 DDIM steps at ~10ms/step = 40ms/frame = 25 FPS (sans VAE).

CURRENT STATE (single chip, 39ms/step):
  - Weight DRAM reads dominate: ~42MB/sub-block x 32 = 1.34GB/step
  - All activations and biases in L1 already
  - SPMD trace execution with zero dispatch overhead

ARCHITECTURE: Megatron-style Column/Row Tensor Parallelism
==========================================================

All chips run the SAME program. The only difference is which weight
slices are loaded at init time. TT-Lang kernels have no awareness of
multi-chip -- they see their local tensors on a 2D grid as usual.

Weight Partitioning (N = number of chips):

  Column-parallel (output dimension sharded):
    qkv_full_w: (1024, 5120) -> each chip gets (1024, 5120/N)
    fc1_w:      (1024, 4096) -> each chip gets (1024, 4096/N)
    adaln_w:    (1024, 6144) -> each chip gets (1024, 6144/N)

  Row-parallel (input dimension sharded):
    out_w:      (1024, 1024) -> each chip gets (1024/N, 1024)
    fc2_w:      (4096, 1024) -> each chip gets (4096/N, 1024)

  Replicated (same on every chip):
    All biases, LN weights, RoPE tables, scaler, mean_scale
    All activations (small: 640KB each)

COMMUNICATION POINTS PER SUB-BLOCK
===================================

Three collectives per sub-block, 32 sub-blocks per step = 96 total.

1. ALL-GATHER after SDPA (before O proj)
   - Each chip has N_HEADS/N heads of attention output
   - Chip k holds: attn_out[:, k*H/N:(k+1)*H/N] where H = D_HEAD * N_HEADS
   - Need full (SEQ, D_MODEL) on all chips for row-parallel O proj
   - Data: each chip sends (SEQ, D_MODEL/N) = 640KB/N, receives full 640KB
   - API: ttnn.all_gather(attn_out, dim=1)

2. ALL-REDUCE after O proj
   - Each chip computed partial O proj from its weight rows
   - Need sum across chips
   - Data: 640KB per chip
   - API: ttnn.all_reduce(o_proj_partial)

3. ALL-REDUCE after FC2
   - Same pattern: each chip computed partial FC2 from its weight rows
   - Data: 640KB per chip
   - API: ttnn.all_reduce(fc2_partial)

Note: FC1 is column-parallel so its output is naturally sharded (each chip
has 1/N of the columns). GELU is elementwise so it stays local. FC2 is
row-parallel and consumes the sharded GELU output directly -- no
communication needed between FC1 and FC2.

TENSOR SETUP (using ttnn mesh APIs)
====================================

mesh_device = ttnn.open_mesh_device(mesh_shape=(1, N))

# Activations: replicated (all chips have identical copies)
x = ttnn.from_torch(x_torch, device=mesh_device,
    mesh_mapper=ttnn.ReplicateTensorToMesh(mesh_device))

# Column-parallel weights: shard along output dim (dim=1)
qkv_w = ttnn.from_torch(qkv_w_torch, device=mesh_device,
    mesh_mapper=ttnn.ShardTensorToMesh(mesh_device, dim=1))

fc1_w = ttnn.from_torch(fc1_w_torch, device=mesh_device,
    mesh_mapper=ttnn.ShardTensorToMesh(mesh_device, dim=1))

adaln_w = ttnn.from_torch(adaln_w_torch, device=mesh_device,
    mesh_mapper=ttnn.ShardTensorToMesh(mesh_device, dim=1))

# Row-parallel weights: shard along input dim (dim=0)
out_w = ttnn.from_torch(out_w_torch, device=mesh_device,
    mesh_mapper=ttnn.ShardTensorToMesh(mesh_device, dim=0))

fc2_w = ttnn.from_torch(fc2_w_torch, device=mesh_device,
    mesh_mapper=ttnn.ShardTensorToMesh(mesh_device, dim=0))

# Biases: replicated
out_b = ttnn.from_torch(out_b_torch, device=mesh_device,
    mesh_mapper=ttnn.ReplicateTensorToMesh(mesh_device))

SUB-BLOCK EXECUTION FLOW
=========================

Each chip executes identically (SPMD). ttnn ops on mesh tensors
automatically dispatch to all chips. Collectives synchronize.

def run_sub_block_tp(prefix, x, dev, scr, mesh_device, ...):
    # --- All local, no communication ---
    norm_mod(x, ...)                      # LN + adaLN modulate
    qkv = ttnn.matmul(modulated, qkv_w)  # column-parallel: each chip 1/N cols

    # --- TT-Lang RoPE kernel: each chip handles N_HEADS/N heads ---
    # rope_layout_spatial sees smaller tensors: (SEQ, 5*D_MODEL/N)
    # Produces q/k/v with N_HEADS/N heads each
    rope_layout(qkv, cos, sin, q, k, v)

    # --- SDPA: each chip processes its heads locally ---
    attn_out = ttnn.sdpa(q, k, v)  # (BATCH_S/N, 1, N_PATCH_PAD, D_HEAD)

    # --- COMMUNICATION POINT 1: all-gather attention output ---
    attn_full = ttnn.all_gather(attn_out, dim=0)  # now all chips have full attn

    # Reshape to (SEQ, D_MODEL)
    attn_2d = reshape(attn_full)

    # --- O proj: row-parallel matmul + all-reduce ---
    o_partial = ttnn.matmul(attn_2d, out_w)  # each chip: partial sum
    # --- COMMUNICATION POINT 2 ---
    o_proj = ttnn.all_reduce(o_partial)
    ttnn.add(o_proj, out_b)

    # --- All local ---
    gated_res_ln_mod(x, o_proj, ...)

    # --- FC1: column-parallel (no comm needed) ---
    fc1 = ttnn.matmul(modulated, fc1_w)  # each chip: 1/N output cols
    ttnn.add(fc1, fc1_b_shard)           # bias also sharded
    gelu(fc1)

    # --- FC2: row-parallel + all-reduce ---
    fc2_partial = ttnn.matmul(fc1, fc2_w)  # each chip: partial sum
    # --- COMMUNICATION POINT 3 ---
    fc2 = ttnn.all_reduce(fc2_partial)
    ttnn.add(fc2, fc2_b)
    gated_residual(z, fc2, gate)

TT-LANG KERNEL CHANGES
=======================

TT-Lang kernels are unaware of multi-chip. They see local tensors only.
Changes needed:

1. rope_layout_spatial: currently parameterized with n_heads_val=16.
   With N chips: n_heads_val=16/N. The kernel already handles arbitrary
   n_heads_val so this just changes the factory parameter.

2. rope_temporal: same change, n_heads_val=16/N.

3. pipe_fused_gated_res_ln_adaln: NO CHANGE. Operates on (SEQ, D_MODEL)
   which is replicated on all chips. The pipe topology is intra-chip only.

4. gated_residual_kernel: NO CHANGE. Elementwise on replicated tensors.

5. All other TT-Lang kernels: NO CHANGE.

The key insight: TT-Lang handles intra-chip parallelism (core-level).
ttnn handles inter-chip parallelism (collective ops). They compose cleanly.

BIAS HANDLING FOR COLUMN-PARALLEL
==================================

Column-parallel matmuls (QKV, FC1, adaLN) produce sharded outputs.
The corresponding biases must also be sharded to match:

  fc1_b: shard along dim=1 to match fc1_w column sharding
  adaln_b: shard along dim=1

Row-parallel matmuls (O proj, FC2) produce partial sums that get
all-reduced. The bias is added AFTER the all-reduce, so it's replicated.

SCRATCH BUFFER CHANGES
=======================

Column-parallel scratch (sharded, 1/N size per chip):
  qkv_full:    (SEQ, 5*D_MODEL/N) instead of (SEQ, 5*D_MODEL)
  fc1:         (SEQ, D_MLP/N)
  gelu:        (SEQ, D_MLP/N)
  q/k/v_sdpa:  1/N the current size
  adaln_packed: (SEQ, 6*D_MODEL/N)

Replicated scratch (same size, all chips):
  modulated, o_proj, fc2, z_scratch, z_a: (SEQ, D_MODEL)

This REDUCES per-chip L1 pressure, giving more room for the sharded
weight tensors that are now 1/N size.

TRACE COMPATIBILITY
====================

The trace captures one DDIM step. With tensor parallelism:
- ttnn.matmul on mesh tensors: traced, dispatches to all chips
- ttnn.all_gather / ttnn.all_reduce: must be trace-compatible
  (verify: these are ttnn ops so should work in trace)
- TT-Lang kernels: traced as before, each chip runs independently
- copy_host_to_device_tensor for DDIM coeff updates: works on mesh

IMPLEMENTATION PHASES
=====================

Phase 1: Weight sharding + ttnn matmul (no TT-Lang changes)
  - Open mesh_device instead of single device
  - Shard weights with ShardTensorToMesh at load time
  - Replicate activations with ReplicateTensorToMesh
  - Insert all_gather and all_reduce at the 3 communication points
  - Keep all TT-Lang kernels identical (they see smaller local tensors)
  - Test correctness first, then performance

Phase 2: Verify TT-Lang kernels work on mesh
  - rope_layout with n_heads_val=16/N
  - pipe_fused_ln on replicated tensors
  - Verify trace capture works on mesh

Phase 3: Optimize communication
  - Overlap all-reduce with subsequent compute if possible
  - Tune num_links parameter for all_gather/all_reduce
  - Profile communication vs compute balance
  - Consider reduce_scatter + all_gather instead of all_reduce
    (avoids full tensor on every chip for FC2 path)

Phase 4: Benchmark and tune
  - Target: 4 chips, 4 DDIM steps, <40ms/frame (>25 FPS)
  - Profile: is communication or compute the bottleneck?
  - If comm-bound: reduce collective data or overlap
  - If compute-bound: already at single-chip limit

EXPECTED PERFORMANCE (4 chips, 4 DDIM steps)
=============================================

Per-step weight reads: 1.34GB / 4 = 336MB per chip
Weight read time @ 200GB/s: ~1.7ms
All-reduce + all-gather: 96 collectives x ~640KB = ~61MB
Communication time @ 50GB/s: ~1.2ms
Compute (elementwise, LN, etc): ~2ms (unchanged)
Estimated per-step: ~8-12ms
4 DDIM steps: ~32-48ms per frame = 21-31 FPS sans VAE
